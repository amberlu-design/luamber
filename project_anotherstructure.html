Instagram Effect Gallery Redesign





Overview

This project looks into Instagram's effect gallery's inefficiecy and reimagines the functionality and intuitiveness of the function.

Scope: Brainstorming & Wireframing, User Research, UI/UX Design, Prototyping

Duration: 2 Weeks



Understanding the Problem

The release of effects has really changed the way users use the app. I noticed a dramatic increase in stories posted because there is a lot more creative freedom for the users. However, the effects that I have saved are only from my friends’ stories. There is also a gallery that contains all the effects for users to search and discover but it is very hard to find and doesn’t allow users to easily find what they are looking for.



User Research

For my user research, I sent out surveys and conducted in-person observations on 10 participants. I reason why I decided to use both of these methods is that surveys are a great way to gather a large amount of data but also the questions prompt them areas of concern that they could miss during an in-person observation. For in-person observations, I am able to get a clear picture of the user journey and emotions and to account for things that they didn’t express on the survey.



Key Findings + Pain Points

People use effects to make their stories look more interesting/2 out of 5 people don’t know where the gallery is located/People who know where it is, don’t browse it often/The function is supposed to be instant but picking filters takes time


Users' Goals

Easy access of gallery and discover more effects to save to collection.

Instagram's Goals

More account interaction, want users to discover effect from friends' stories.



Design Goals: How Might We?

How might we improve the effect gallery's function and process?



User Personas



Redesign

Problem 1

Low discoverability of effects gallery


Redesign Ideas

Design A

Only displays 4 buttons and "looks like" it takes up more screen space. However, more considerate of the human anatomy as most users swipe through using their thumb.


Design B

Wouldn't work because it clashes with the "cross" that comes up when users are scrolling through different effects. Also, when an effect has multiple versions, they come up there.



Design C

Doesn't make unnecessary changes, also solves the problem for users to get to the library more easily.




Solution



I decided that Design C is the best solution. It just relocates the effect gallery next to home; it doesn't make unnecessary changes, so the users won't be confused and solves the problem of easy access to the library.






Problem 2

Poor system and user intuitiveness for browsing effects


Redesign Ideas

Design A

3 columns are too clusted for the screen. Although displays more effects in one viewing, makes it harder to properly view each effect's thumbnail.


Design B

2 columns instead of 3. It displays the thumbnails with a shorter height, but still enough for the users to view each effects properly. The height of each effect's display is reduced to fit more in one view. This reduces the amount of actions needed by the users.



Solution


when different "categories" are selected, the current page refreshes instead of sliding into new window to reduce noisy transitions. Users can also swipe to get to different categories.




Quick save button. Currently, users have to go into the effect to be able to save it. Adding a quick save option on the main browsing screen will increase convenience and speed of the process.
